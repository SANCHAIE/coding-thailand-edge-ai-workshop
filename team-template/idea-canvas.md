# 💡 Idea Canvas — Lean Canvas (4 ช่องแรก)

> ทำช่วงท้ายวัน เปลี่ยน "ของที่รันได้วันนี้" → "ไอเดียนวัตกรรมที่ต่อยอด Day 2 ได้"
> เริ่มจาก **ปัญหา** ไม่ใช่ "เรามี sensor อะไร" · ทำ 4 ช่องนี้ให้คมก่อน ที่เหลือไว้ทีหลัง

- **ชื่อ Prototype (ชั่วคราว):** MotionSense

---

## 1️⃣ Problem — ปัญหาคืออะไร

เขียน 2–3 ข้อ ที่มีคน "ลำบาก / เสียเวลา / พลาด" จริง:
```
- คนที่กำลังเครียดหรือโกรธมักไม่รู้ตัวว่าพฤติกรรมการเคลื่อนไหวของตัวเองเปลี่ยนไป
- การสังเกตอารมณ์ด้วยสายตามนุษย์ไม่แม่นยำและไม่ต่อเนื่องในบางเหตุการ
- ผู้ปกครองหรือครูไม่สามารถสังเกตอารมณ์ของเด็กทุกคนได้ตลอดเวลา
```
**ตอนนี้คนมแก้ปัญหานี้ยังไง (ของเดิม)?** สังเกตจากสีหน้าและพฤติกรรมพูดคุยสอบถาม,ไม่มีระบบแจ้งเตือนแบบอัตโนมัติ

## 2️⃣ Customer — ใครคือผู้ใช้จริง

เจาะให้ชัด ไม่ใช่ "ทุกคน":
```
นักเรียน อายุ 15–22 ปีและครูหรืออาจารย์ที่ดูแลนักเรียน
```

## 3️⃣ Value (USP) — คุณค่าเด่นในประโยคเดียว

```
"MotionSense ช่วยให้ครูและผู้ปกครองตรวจจับพฤติกรรมการเคลื่อนไหวที่บ่งบอกถึงความเครียดหรืออารมณ์ผิดปกติได้แบบ Real-time แตกต่างจากการสังเกตด้วยสายตา เพราะใช้ Edge AI วิเคราะห์อัตโนมัติบนอุปกรณ์"
```
## 4️⃣ Solution — แก้ด้วยอะไร

- **Input ที่ใช้:** Movement (IMU) 
- **Class ที่ AI แยก:** 1. หน้าปกติ 2.ยิ้ม 3.มุมปากตก 4.ขมวดคิ้ว 5.โกรธ
- **Output ตอบสนอง:** Pixels, Buzzer
---

## 🎯 MVP — ฟังก์ชันเดียวที่ต้องสาธิตให้ได้

```
Input → IMU → AI Model → Arduino UNO Q → Prediction → LED + Buzzer
```

**1 core function ที่จะโชว์ Day 2/3:** เมื่อ AI ตรวจพบรูปแบบการเคลื่อนไหวที่เข้าข่าย "เครียด" หรือ "โกรธ" จะเปลี่ยนไฟ LED เป็นสีแดงและส่งเสียง Buzzer แจ้งเตือน

> คาถา: AI ไม่ได้อ่านอารมณ์ แต่วิเคราะห์รูปแบบการเคลื่อนไหวของร่างกาย และประเมินพฤติกรรมที่มีความสัมพันธ์กับอารมณ์โกรธ
