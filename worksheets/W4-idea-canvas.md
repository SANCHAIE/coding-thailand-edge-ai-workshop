## 🎯 1. Problem & User

### a) เราจะแก้ปัญหาอะไร?
```
เราต้องการแก้ปัญหาการบุกรุกพื้นที่ส่วนบุคคล/พื้นที่หวงห้าม และระบบคัดกรองความปลอดภัยที่ล่าช้า
รวมถึงการรั่วไหลของข้อมูลภาพใบหน้าของบุคลากรหากต้องส่งไปประมวลผลบนระบบ Cloud
โดยระบบนี้จะช่วยคัดกรองใบหน้า แจ้งเตือน และต้อนรับได้ทันทีแบบ On-device 100%
```

### b) ใครคือ user หลัก?
```
นิติบุคคลอาคารชุด/หมู่บ้านจัดสรร ที่ต้องการคัดกรองคนเข้า-ออกบริเวณป้อมยาม

เจ้าของร้านค้า/ร้านสะดวกซื้อขนาดเล็ก ที่ต้องการระบบเตือนภัยบุคคลเฝ้าระวัง (Blacklist) ในร้าน

เจ้าของบ้านพักอาศัยอัจฉริยะ (Smart Home) ที่ให้ความสำคัญกับเรื่อง Data Privacy (ภาพไม่หลุดออกนอกบ้าน)
```

### c) ทำไม Edge AI ถึงเหมาะกับปัญหานี้? (ทำไมไม่ใช่ Cloud)
```
[เขียน]

เช่น: ต้องการ privacy / ไม่มีเน็ตในชนบท / ต้อง response < 1 วินาที
```

### Unique Value (USP)

```

ต่างจาก [solution อื่น] เพราะ [ข้อดี]
```

ตัวอย่าง:
```
GuardianEye ช่วย นิติบุคคลและเจ้าของบ้าน คัดกรองและแจ้งเตือนบุคคลเข้าพื้นที่แบบเรียลไทม์
ต่างจาก กล้องวงจรปิดทั่วไปและระบบ Cloud AI เพราะ ประมวลผลในตัวโดยไม่ต้องต่ออินเทอร์เน็ต รักษาความเป็นส่วนตัวสูงสุด และแจ้งเตือนด้วยเสียงจำแนกตามบุคคลทันที
```


## 🎬 4. Scenario ตอนโชว์การทำงาน

ถ้าต้องโชว์การทำงานภายใน 2 นาที จะเรียงขั้นตอนอย่างไร?

| ขั้นตอน | ทำอะไร | เวลา |
|---|---|---|
| 1 | **Introduction:** แนะนำปัญหาเรื่องความปลอดภัย และ Data Privacy ของกล้องทั่วไป พร้อมโชว์ตัวกล่องอุปกรณ์ | 20 วินาที |
| 2 | **Staff Demo:** สมาชิกในทีม (Staff) เดินผ่านกล้อง -> ไฟ Pixels ขึ้นสีเขียว + ลำโพงกล่าว "ยินดีต้อนรับค่ะ" | 30 วินาที |
| 3 | **Stranger & Blacklist Demo:** ให้เพื่อนต่างทีมเดินผ่าน -> ไฟขึ้นสีเหลืองเตือน และลองเอาภาพถ่ายบนมือถือมาส่องหลอก -> ระบบตรวจจับได้ว่าเป็นความเสี่ยง ส่งไฟสีแดงพร้อมเปิดเสียงไซเรน | 50 วินาที |
| 4 | **Conclusion:** สรุป Metrics เรื่อง Response Time (<50ms) และประโยชน์เชิงพาณิชย์ | 20 วินาที |

## 📊 6. Success Metrics

ในรอบสรุปงาน ทีมจะถือว่า prototype สำเร็จเมื่อ:

- [x] ✅ Functionality: สามารถตรวจจับ คัดกรองใบหน้า และสั่งการเปิดไฟล์เสียงพากย์แจ้งเตือนออกลำโพงได้ถูกต้องตามเงื่อนไขแยกขาดทั้ง 4 คลาส
- [x] ✅ Accuracy: 85% ขึ้นไปในสภาวะแสงใช้งานจริงภายในอาคาร
- [x] ✅ Response time: ภายใน 60 ms ต่อเฟรมภาพ
- [x] ✅ ทดสอบจริงในห้องแล้วทำงานได้โดยไม่เกิดอาการระบบค้างหรือ Memory Leak เมื่อเปิดทิ้งไว้นานกว่า 15 นาที
- [x] ✅ User เข้าใจในการดูครั้งแรก: ผู้ประเมินสามารถเข้าใจสถานะความปลอดภัยได้ทันทีจากสีของไฟ Pixels โดยไม่ต้องมองหน้าจอ Monitor

## 🚧 7. Risk & Mitigation

### Top 3 ความเสี่ยง

| ความเสี่ยง | ผลกระทบ | วิธีแก้ |
|---|---|---|
| สภาพแสงเปลี่ยนช่วงเย็น/ปิดไฟ | โมเดลจำแนกใบหน้าผิดพลาด หรือมองไม่เห็น (No_Face) | ติดตั้งชุดไฟ LED ขนาดเล็กเป็น Ring Light หน้ากล้องเพื่อชดเชยแสงโดยอัตโนมัติ |
| การโจมตีระบบด้วยภาพถ่าย (Spoofing) | คนนอกใช้ภาพถ่ายสตาฟหลอกผ่านประตูเข้ามาได้ | เพิ่ม Logic การเช็ก Landmark ใบหน้าว่าต้องมีการขยับหรือกะพริบตาก่อนปลดล็อค |
| บอร์ดมีความร้อนสูงเมื่อรันโมเดลต่อเนื่อง | เฟรมเรตตก (Lag) หรือระบบชัตดาวน์ตัวเอง | ออกแบบเคส Form Factor ให้มีช่องระบายอากาศคู่และติดตั้งฮีทซิงค์ (Heatsink) บนชิปประมวลผล |
